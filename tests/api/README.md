# API Test Framework

Scalable REST API test suite using **Python 3.13**, **Pytest**, **HTTPX**, and **Pydantic v2**. Covers GET, POST, PUT, DELETE verbs with full schema validation and a session-scoped authenticated client fixture.

## Prerequisites

- Python 3.13

## Setup

```bash
cd tests/api

python3.13 -m venv .venv
source .venv/bin/activate    # Windows: .venv\Scripts\activate

pip install -r requirements-api.txt
```

## Configuration

All settings are configured via environment variables with sensible defaults:

| Variable         | Default                  | Description                          |
|------------------|--------------------------|--------------------------------------|
| `API_BASE_URL`   | `http://localhost:80`    | Base URL of the API server           |
| `API_USERNAME`   | `admin`                  | Username for auth token acquisition  |
| `API_PASSWORD`   | `password`               | Password for auth token acquisition  |
| `API_AUTH_PATH`  | `/api/auth/login`        | Auth login endpoint path             |
| `API_TIMEOUT`    | `10`                     | HTTP request timeout (seconds)       |

## Running Tests

### Smoke tests only (health endpoint ‚Äî always available)
```bash
pytest tests/ -m smoke
```

### All tests against a local server
```bash
API_BASE_URL=http://localhost:80 pytest tests/
```

### Specific test file
```bash
pytest tests/test_health.py -v
```

### Parallel execution
```bash
pytest tests/ -n auto
```

### With HTML report
Reports are generated by default to `reports/html/report.html` and `reports/junit/results.xml`.

### Skip unavailable endpoints
Tests for `auth`, `users`, and `products` endpoints automatically `pytest.skip()` when the
target server returns 404 for those paths. Health tests always run.

## Test Marks

| Mark        | Description                                   |
|-------------|-----------------------------------------------|
| `smoke`     | Critical path ‚Äî health endpoint only          |
| `regression`| Full regression suite                         |
| `api`       | All API tests                                 |
| `health`    | Health endpoint tests                         |
| `auth`      | Auth / login tests                            |
| `users`     | User CRUD tests                               |
| `products`  | Product CRUD tests                            |

## Fixture Design

### `auth_token` (session-scoped)
Attempts a real login via `POST /api/auth/login` once per session. If the endpoint
is unavailable or returns an error, returns `None` ‚Äî auth-dependent tests skip gracefully.

### `api_client` (session-scoped)
An `httpx.Client` pre-configured with:
- `base_url` from `API_BASE_URL`
- `Authorization: Bearer <token>` header (when token available)
- Default timeout
- `Accept: application/json` header

### `unauth_client` (function-scoped)
An `httpx.Client` with NO Authorization header ‚Äî used for 401 negative tests.
Function-scoped to prevent state leakage between tests.

## Schema Validation

Every response is validated against a Pydantic v2 model via `assert_schema()`:

```python
response = api_client.get("/api/healthz")
assert_status(response, 200)
data = assert_schema(response, HealthStatusResponse)
assert data.status == "ok"
```
